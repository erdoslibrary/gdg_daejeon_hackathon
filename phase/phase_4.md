# Phase 4: 위트 있는 디테일 마무리 ✨

## 이 단계에서 한 일

전체 앱에 **마감 터치**를 더했습니다. 사용자 경험을 풍부하게 만드는 미세한 인터랙션과 위트 있는 디테일입니다.

## 추가된 디테일

### 1. 🎉 입장 시스템 메시지

유저가 닉네임을 입력하고 주방에 입장하면, 채팅에 자동으로 시스템 메시지가 뜹니다:

> 🎉 '집밥요정'님이 주방에 입장했습니다!

이 메시지는 **1회만** 전송됩니다 (`useRef`로 중복 방지):

```typescript
const entryMessageSent = useRef(false);

useEffect(() => {
  if (nickname && !entryMessageSent.current) {
    entryMessageSent.current = true;
    // 시스템 메시지 전송
  }
}, [nickname]);
```

### 2. 🍚 식사 모드 톤 변화

유저가 **[먹는 중]** 상태로 변경하면, 화면 전체에 따뜻한 앰버 톤 오버레이가 부드럽게 적용됩니다:

```tsx
{isEatingMode && (
  <motion.div
    initial={{ opacity: 0 }}
    animate={{ opacity: 1 }}
    className="fixed inset-0 bg-amber-900/5 pointer-events-none"
  />
)}
```

- `pointer-events-none`: 오버레이가 클릭을 방해하지 않음
- `bg-amber-900/5`: 매우 미묘한 따뜻한 톤 (5% 불투명도)
- 상태가 바뀌면 자동으로 사라짐

### 3. 🧽 설거지 대기 흑백 필터

**[설거지 대기]** 상태의 유저 칩은 **완전한 흑백(grayscale)** 처리됩니다.

본인 칩도 예외 없이 흑백 처리되어 "설거지는 피할 수 없다"는 느낌을 줍니다:

```typescript
// 본인 + 설거지 상태
"bg-surface-dim text-on-surface-variant font-bold ring-1 ring-outline/30 grayscale"

// 다른 유저 + 설거지 상태
"bg-surface-dim text-on-surface-variant grayscale opacity-70"
```

### 4. 📊 조리 진행률 바

조리 순서에 **진행률 바**와 **완료 카운터**를 추가했습니다:

- 상단에 `2/4 완료` 뱃지 표시
- 올리브 그린 프로그레스 바가 부드럽게 채워짐
- **4/4 전체 완료** 시 축하 메시지 등장:
  > 🎉 요리 완성! 이제 맛있게 드세요~

## 핵심 개념 설명

### useRef란?

`useState`와 비슷하지만 **값이 바뀌어도 화면을 다시 그리지 않는** 저장소입니다:

| | useState | useRef |
|---|---|---|
| 값 변경 시 화면 리렌더링 | ✅ 리렌더링 | ❌ 리렌더링 없음 |
| 주 사용처 | UI에 보이는 데이터 | 내부 추적용 값 |
| 예시 | 닉네임, 메시지 목록 | "이미 입장 메시지를 보냈는가?" |

### pointer-events-none이란?

CSS 속성으로, 해당 요소를 **마우스 클릭/터치가 통과**하게 만듭니다.

화면 전체를 덮는 톤 오버레이가 있지만, 사용자는 그 아래의 버튼을 정상적으로 클릭할 수 있습니다.

1. **입장**: 닉네임 설정 → "오늘의 메뉴(김치볶음밥)" 확인
2. **요리**: [요리 중] → 레시피 따라하며 체크 → 진행률 바 채워짐
3. **소통**: 실시간 채팅으로 다른 유저와 대화
4. **완료**: [먹는 중] 클릭 → 따뜻한 톤 + 시스템 알림
5. **마무리**: [설거지 대기] → 흑백 필터 + 위트 문구
